# 🧹 Memory Compaction Prompt (Heartbeat / Weekly Review)

## 🎯 任务
处理记忆缓冲区，升级高频项，丢弃噪声，保持长期记忆纯净。

## ⚖️ 裁决规则
1. **权威优先**：`user_direct` (1.0) > `interaction_log` (0.8) > `daily_summary` (0.6)
2. **时效覆盖**：同源/同实体冲突时，日期较新且权威度 ≥ 旧条目的自动覆盖旧条目。
3. **遗忘机制**：连续 14 天未更新且标记为 `routine` 的条目，标记 `archived: true`。

## 🔄 操作指令
1. 读取当前 YAML 列表与缓冲区。
2. **Promote (升级)**：缓冲区中连续 3 天出现或权威度 ≥ 0.7 的观察，转化为正式 YAML 条目。
3. **Evict (清理)**：删除过时、矛盾或低价值记录。
4. **Compact (压缩)**：若缓冲区 > 500 字，提炼核心假设，丢弃原始日志。
5. 输出更新后的完整 YAML 头部 + 精简缓冲区。

## ⚠️ P0 实体隔离检查
输出前自检：
- 是否所有 `entity` 字段均为全名？
- 是否混入了“对方/这人/她”等代词？
- 冲突裁决是否严格遵循权威度 > 日期 顺序？

确认无误后输出最终文件内容。
